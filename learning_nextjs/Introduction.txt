1. NextJS is a framework that uses React to build frontend applications.

2. Even though React has good capable of building applications it mainly focus on UI layer.

3. NextJS has everything ready that is helpful to build a production ready applications such as routing, data fetching etc.

4. You no need to install any additional packages.

Features of NextJS:

1. Routing. we need not to install any other packages for routing. NextJS has file based Routing.

2. API Routes. we can be able to build api's simulataneously and get connected to backend.

3. Rendering. NextJS supports both Client Side Rendering and Server Side Rendering making more optimization.

4. Data fetching. we need to install separate library for data fetching. NextJS has its own.

5. Styling. NextJS supports normal css and tailwind css.

6. Dev and prob build system. 

Installation:

npx create-next-app@latest my-app

cd my-app

npm run dev

React Server Components(RSC):

1. React server Components is a new architecture that was introduced by React and quicky adopted by NextJS.

2. In this architecture it creates react components by dividing them into two distinct types:

    a. Server Components
    b. Client Components

Server Components:

1. By Default, NextJS treats all the components as Server Components.

2. These components can perform server-side task like reading files and fetching data from DB.

3. But they cannot use React hooks and User Interactions.

Client Components:

1. To create a Client components we need to add the "use client" directive at the top of your component file.

2. These components cannot perform server side tasks, but can use React hooks and user interactions.
